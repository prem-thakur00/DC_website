🎯 Characteristics of Data Communication
Definition: Data communication is the exchange of data between two devices via some form of transmission medium such as a wire cable. The exchanged data can be text, images, audio, video, or multimedia files.
📦 Delivery Success Rate (%) 100
✅ Data Accuracy (% Correct) 100
⏱️ Transmission Delay (ms) 0
📊 Jitter Variation (ms) 0
📤 Delivery
The system must deliver data to the exact destination with security protection. No other receiver should receive the data unintentionally.
✅ Accuracy
Data must be received without alteration or damage. The receiver gets the exact same data as sent by the sender, maintaining data integrity.
⏰ Timeliness
Data must be delivered in a timely manner. Delayed delivery can make the data useless to the receiver. Critical for real-time applications.
📈 Jitter
Variation in packet arrival time. Uneven delays cause quality issues in audio/video (e.g., packets arriving 30ms, then 40ms, affecting stream quality).
🌐 Network Topologies
Definition: Network topology refers to the physical arrangement of devices and connections in a network. It determines how data flows and how devices communicate with each other.
Point-to-Point: Direct connection between exactly 2 devices. Communication is exclusive between these two devices only. Simplest form of connection.
Mesh Topology: Every device connects to every other device. Maximum redundancy and fault tolerance. Uses most cabling but highest reliability. Suitable for critical networks.
Star Topology: All devices connect through a central hub/switch. Central device manages all communication. Easy to manage, single point of failure, moderate cabling.
Bus Topology: All devices share a single communication line. Devices tap into the bus with drop lines. Simple but prone to collisions. Used in older Ethernet systems.
Ring Topology: Devices form a circular loop. Data travels in one direction. Token-based access. Good performance under heavy load. One break breaks the ring.

📊 Transmission Modes
Definition: Transmission modes define the direction in which data can flow between two devices. There are three main types based on data flow directionality.
Simplex Mode: Unidirectional communication. Data flows in only ONE direction, always from sender to receiver. Receiver CANNOT send data back.
Half-Duplex Mode: Bidirectional but NOT simultaneously. Devices can both send and receive, but NOT at the same time. Take turns communicating.
Full-Duplex Mode: Bidirectional and SIMULTANEOUS communication. Devices can send AND receive data at the exact same time without interference.
Mode Direction Simultaneous Bandwidth Used Latency Example
Simplex One-way N/A Full Low TV, Radio
Half-Duplex Both Ways No Full (per direction) Higher Walkie-Talkie
Full-Duplex Both Ways Yes Shared (both directions) Lowest Phone, Video Call
📡 Communication Channels
Definition: A communication channel (or transmission medium) is the physical path through which data is transmitted from sender to receiver. Channels are classified as Guided (wired) or Unguided (wireless).
🌡️ Environmental Interference Level 0%
📶 Signal Strength 100%

🔗 Guided Media (Wired)

1️⃣ Twisted Pair Cable
Two copper wires twisted together to reduce electromagnetic interference. Most common in LANs. Affordable but limited range (~100m).
📊 Specs: Speed: 10-1000 Mbps | Cost: Low | Range: 100m
2️⃣ Coaxial Cable
Central copper core surrounded by insulation and outer conductor. Higher bandwidth than twisted pair. Used in cable TV and older networks.
📊 Specs: Speed: 10-1000 Mbps | Cost: Medium | Range: 500m
3️⃣ Fiber Optic Cable
Uses light pulses to transmit data through glass fibers. Highest bandwidth, immune to electromagnetic interference, very fast but expensive.
📊 Specs: Speed: Gbps | Cost: High | Range: km

📶 Unguided Media (Wireless)

1️⃣ Radio Waves
Electromagnetic waves in lower frequency range. Used for AM/FM radio and long-distance communication. Good range, affected by obstacles.
📊 Specs: Speed: Mbps | Cost: Low | Range: km
2️⃣ Microwave
Higher frequency electromagnetic waves. Used for satellite and cellular communication. Good bandwidth, needs line-of-sight.
📊 Specs: Speed: Gbps | Cost: Medium | Range: 100km
3️⃣ Infrared Waves
Very short range, high-frequency waves. Used for remote controls and short-distance wireless. Cannot penetrate walls. Very fast.
📊 Specs: Speed: Gbps | Cost: Low | Range: 10m
🏛️ OSI (Open Systems Interconnection) Model
Definition: The OSI model is a 7-layer conceptual framework that standardizes how data communication occurs across different devices and technologies. It divides network functions into distinct layers, each with specific responsibilities.
💡 Key Concept: Each layer provides services to the layer above it and uses services from the layer below. Data encapsulation occurs as it moves down the layers (sender side) and decapsulation as it moves up (receiver side).
7
Application Layer
Function: User interface and applications
Examples: HTTP, FTP, SMTP, POP3, Telnet
Devices: Browsers, Email clients
PDU: Data/Message
6
Presentation Layer
Function: Data translation, encryption, compression
Examples: SSL/TLS, JPEG, MPEG, ASCII
Task: Convert data into presentable form
PDU: Data
5
Session Layer
Function: Manages communication sessions/connections
Examples: RPC, PPTP, L2TP
Task: Establish, maintain, terminate sessions
PDU: Data
4
Transport Layer
Function: End-to-end delivery, flow control, error handling
Examples: TCP, UDP, SCTP
Task: Break data into segments, ensure reliable delivery
PDU: Segment/Datagram
3
Network Layer
Function: Routing, logical addressing, forwarding
Examples: IP (IPv4, IPv6), ICMP, ARP, IGMP
Task: Move packets between networks
PDU: Packet
2
Data Link Layer
Function: Error detection, MAC addressing, framing
Examples: Ethernet, Wi-Fi (802.11), PPP, MAC
Task: Reliable hop-to-hop delivery
PDU: Frame
1
Physical Layer
Function: Raw bit transmission, physical mediums
Examples: Cables, Hub, Repeater, Modem, Fiber
Task: Send binary signals over physical medium
PDU: Bit
🌍 TCP/IP Model (Internet Protocol Suite)
Definition: The TCP/IP model is a 4-layer practical model that forms the foundation of the modern Internet. It's simpler than OSI and combines multiple OSI layers into each TCP/IP layer. TCP (Transmission Control Protocol) and IP (Internet Protocol) are its core protocols.

Layers of TCP/IP

4️⃣ Application Layer
Protocols: HTTP, HTTPS, FTP, SMTP, DNS, Telnet, SSH, SNMP
Function: User applications and services
Combines OSI Layers: 5, 6, 7
3️⃣ Transport Layer
Protocols: TCP, UDP, SCTP
Function: End-to-end communication & reliability
Same as OSI Layer: 4
2️⃣ Internet Layer
Protocols: IP (IPv4/IPv6), ICMP, ARP, IGMP
Function: Routing & logical addressing
Same as OSI Layer: 3
1️⃣ Network Access (Link) Layer
Protocols: Ethernet, Wi-Fi, PPP, MAC
Function: Physical transmission
Combines OSI Layers: 1, 2

OSI vs TCP/IP

🔄 Key Differences:
• OSI: Theoretical model, 7 layers, detailed, academic standard
• TCP/IP: Practical model, 4 layers, used in real Internet, industry standard
• OSI combines layers 5, 6, 7 into TCP/IP's Application layer
• OSI combines layers 1, 2 into TCP/IP's Network Access layer
