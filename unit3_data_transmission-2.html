Modulation Techniques

Modulation is the process of varying one or more properties of a periodic carrier signal with respect to a modulating message signal.

📖 Definition

Modulation is a technique that changes signal characteristics by altering the amplitude, frequency, or phase of a high-frequency carrier signal in accordance with a low-frequency message signal. This allows information to be transmitted efficiently over long distances through various media.

🎯 Why Modulate?

Without modulation, audio/data signals (low frequency) cannot travel long distances. Carrier signal acts as a "vehicle" for the message.

📡 Carrier Signal

A high-frequency sinusoidal wave: c(t) = Ac·sin(2πfct + φ). Three properties — Amplitude (Ac), Frequency (fc), Phase (φ) — can be modified.

🔁 Analog Modulation

Message is a continuous analog signal. Includes AM, FM, PM. Carrier properties vary continuously with the message signal.

🔢 Digital Modulation

Message is binary (0s and 1s). Includes ASK, FSK, PSK. Each bit or bit group changes amplitude, frequency, or phase of carrier.

Analog Modulation

Amplitude Modulation (AM)

The amplitude of the carrier signal is varied in proportion to the instantaneous amplitude of the message signal.

📖 Definition

Amplitude Modulation (AM) is a type of modulation in which the amplitude of a high-frequency carrier signal is varied according to the instantaneous value of the message signal. The frequency and phase of the carrier remain constant. The envelope of the modulated signal exactly follows the message waveform.

AM(t) = Ac · [1 + μ · cos(2π·fm·t)] · cos(2π·fc·t)
μ (modulation index) = Am / Ac

Analog Modulation

Frequency Modulation (FM)

The instantaneous frequency of the carrier signal is varied in proportion to the message signal amplitude.

📖 Definition

Frequency Modulation (FM) is a type of analog modulation in which the frequency of the carrier signal is varied according to the amplitude of the message signal. The amplitude and initial phase of the carrier remain constant. FM is more resistant to noise than AM and is widely used in high-fidelity broadcasting (88–108 MHz band).

FM(t) = Ac · sin[2π·fc·t + β·sin(2π·fm·t)]
β (modulation index) = Δf / fm

Analog Modulation

Phase Modulation (PM)

The instantaneous phase of the carrier signal is varied in proportion to the message signal.

📖 Definition

Phase Modulation (PM) is a type of analog modulation in which the phase angle of the carrier signal varies proportionally to the message signal. The amplitude and frequency of the carrier remain constant. During positive half-cycles of the message, the carrier phase shifts in one direction; during negative half-cycles, it shifts in the opposite direction. PM is closely related to FM.

PM(t) = Ac · sin[2π·fc·t + kp·m(t)]
kp = phase sensitivity constant (rad/V)

Digital-to-Digital · D2D

Line Coding Simulator

Line coding converts a sequence of bits into a digital signal. Enter any binary sequence and see how different schemes encode it visually.

📖 Definition

Line Coding is the process of converting digital data (bits) into a digital signal for transmission over a physical medium. At the sender, digital data are encoded into signal elements. At the receiver, the digital data are recreated by decoding the received signal. Key characteristics: signal rate vs data rate, bandwidth, DC components, self-synchronization, and error detection.

Signal Rate S = N/r baud  |  Bandwidth B_min = c × N × (1/r)

📌 Signal vs Data Element

One signal element can carry multiple data elements. If r = data elements per signal element, then S = N/r baud.

📉 Baseline Wandering

A long string of 0s or 1s causes receiver's running average to drift — called baseline wandering. Good schemes avoid this.

⚡ DC Component

Constant voltage levels create DC component (near-zero frequency). This causes baseline wandering and cannot pass through AC-coupled links.

🔄 Self-Synchronization

The receiver must know when each bit begins and ends. Manchester encoding provides built-in clock via mid-bit transitions.

Digital-to-Digital · D2D

Block Coding (4B/5B)

Block coding changes a block of m bits into n bits (n > m), adding redundancy for error detection and DC component control.

📖 Definition

Block Coding (mB/nB coding) replaces m-bit groups with n-bit codewords where n > m. In 4B/5B encoding, each 4-bit data group is replaced with a 5-bit code. This gives 2⁵ = 32 possible codes for 2⁴ = 16 data patterns, leaving 16 extra codes for control and synchronization. No code has more than two consecutive zeros, preventing DC components when combined with NRZ-I.

4B/5B: 16 data patterns → mapped to 16 of 32 available 5-bit codes

Digital-to-Digital · D2D

Scrambling (B8ZS)

Scrambling replaces long strings of zeros in a bipolar signal with special patterns that maintain synchronization without adding extra bits.

📖 Definition

Scrambling is a technique used to overcome the DC component and synchronization problems caused by long sequences of 0s in bipolar encoding. Unlike block coding, scrambling does not increase the number of bits. B8ZS (Bipolar 8-Zero Substitution) replaces any sequence of 8 consecutive zeros with the pattern 000VB0VB, where V = Violation pulse (same polarity as last 1) and B = Bipolar pulse (follows AMI rule). HDB3 replaces every 4 consecutive zeros.

B8ZS: 00000000 → 000VB0VB (where V = violation, B = bipolar)

Data Transmission

Modes of Data Transmission

Data can be transmitted in different ways — parallel or serial. Serial transmission can be synchronous or asynchronous.

⚡ Parallel Transmission
Multiple bits sent simultaneously over multiple wires. Fast but expensive for long distances. Used in printers, internal buses (e.g., 8-bit, 16-bit data bus). Costly over long distances due to synchronization issues.
🔗 Serial Transmission
One bit at a time over a single wire. Slower but cheaper for long distances. Used in USB, UART, I²C. Serial transmission is further divided into Asynchronous and Synchronous modes.
📨 Asynchronous Transmission
No shared clock. Each byte (character) is framed with a start bit (0) and stop bit(s) (1). The gap between characters can be any length. Simple, low cost. Used in: UART, RS-232, keyboards. Overhead: 2–3 extra bits per byte.
🕐 Synchronous Transmission
Shared clock signal. Data is sent in continuous blocks/frames without start/stop bits. Faster and more efficient than async. Used in: HDLC, Ethernet, T1/T3 lines, fiber optic systems. Requires precise timing synchronization.
